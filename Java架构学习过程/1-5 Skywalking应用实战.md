# 一、本节课核心主题
学习 **SkyWalking** —— 一款**国产、开源、零代码侵入**的微服务 **APM（应用性能监控）工具**，用于解决微服务架构下**请求链路混乱、问题难定位、性能难排查**的痛点。

---

# 二、为什么需要微服务监控？
1. 微服务拆分后，服务数量多、实例多（多个 Tomcat）
2. 一次用户请求会跨多个服务跳转
3. 出问题时：不知道请求走了哪台机器、日志散落在多台服务器、排查极慢
4. 传统日志排查效率太低，需要**集中监控 + 全链路追踪**

---

# 三、微服务监控三大核心要素
1. **Metrics（度量）**
   QPS、接口调用量、成功率、响应时间、CPU/内存/GC 等指标。

2. **Logs（日志）**
   业务日志、异常日志、请求日志。

3. **Tracing（链路追踪）**
   记录一次请求**完整调用路径**：经过哪些服务、哪个实例、耗时多少。

---

# 四、分布式链路追踪基础
1. **起源**
   Google Dapper 论文 → 催生 OpenTracing 规范。

2. **核心概念**
    - **Trace**：一次完整的请求/事务
    - **Span**：请求过程中的一个调用单元（服务/接口/方法）
    - 包含：开始/结束时间、操作名、父子关系、上下游调用

3. **展示方式**
   时间轴 + 调用树，直观看到串行/并行、耗时分布、瓶颈点。

---

# 五、SkyWalking 是什么
- 国人开源（华为吴晟主导）
- **零代码侵入**，基于 Java Agent 字节码增强
- 集 **Metrics + Tracing + Logs** 于一体
- 支持微服务、云原生、多种中间件

---

# 六、SkyWalking 架构
1. **Agent（探针）**
   部署在应用端，采集数据，无侵入。

2. **OAP Server**
   接收、处理、存储监控数据。

3. **Storage**
   常用 Elasticsearch。

4. **UI**
   可视化控制台。

---

# 七、部署与接入（重点）
## 1. 部署方式
推荐使用 **Docker Compose** 一键部署：
- Elasticsearch
- SkyWalking OAP
- SkyWalking UI

## 2. 项目接入（零侵入）
只需要在 JVM 启动参数添加：
```
-javaagent:skywalking-agent.jar
-Dskywalking.agent.service_name=你的服务名
-Dskywalking.collector.backend_service=oap地址:11800
```
**不用改任何业务代码**。

---

# 八、SkyWalking 核心功能（高频使用）
## 1. 仪表盘（Dashboard）
- 全局服务健康度
- QPS、响应时间、成功率
- 实例监控（CPU、JVM、GC、线程）
- 慢接口统计

## 2. 拓扑图（Topology）
- 上帝视角看**服务调用关系**
- 一目了然：谁调用谁、是否有不合理依赖

## 3. 追踪（Trace）—— 最核心功能
- 查看**单次请求完整链路**
- 显示 TraceID、服务、实例、IP、URL、SQL
- 定位哪一步慢、哪一步报错
- 支持列表、树状、表格视图

## 4. 性能剖析（Profiling）
- 针对**慢接口**
- 自动分析方法调用栈
- 精准定位：哪个类、哪个方法、哪一行代码最耗时

## 5. 日志关联（Logs）
- 在日志格式中加入 `%X{traceId}`
- 可通过 TraceID 一次性捞出**一次请求的所有日志**
- 支持日志上云集中查看

## 6. 告警与事件
- 监控异常、超时、错误率
- 触发告警（课程不重点）

---

# 九、典型使用场景（工作真实流程）
1. 接口响应慢
   → 拓扑看依赖
   → 追踪看哪步耗时
   → 性能剖析定位具体方法

2. 线上报错
   → 拿 TraceID
   → 搜对应服务器日志
   → 快速定位问题

3. 架构梳理
   → 拓扑图看服务调用是否规范
